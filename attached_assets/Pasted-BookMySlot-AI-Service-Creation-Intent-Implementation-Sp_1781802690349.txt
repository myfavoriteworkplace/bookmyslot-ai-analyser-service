BookMySlot AI Service — Creation Intent & Implementation Specification
1. Objective
Create a separate AI microservice called:
bookmyslot-ai-service
Purpose:
To provide AI-powered dental X-ray analysis capability for the existing BookMySlot healthcare application.
The service will:
Receive dental X-ray images
Run inference using the trained YOLOv8 DENTEX model
Detect dental findings
Return structured JSON results
Be consumed by the existing Node.js backend
2. Existing Application Context
Current system:
Frontend
Technology:
React
Responsibility:
Doctor dashboard
Patient management
X-ray upload UI
Display AI analysis results
Backend
Technology:
Node.js
Responsibility:

Appointment data

API gateway
The Node backend should NOT directly run the AI model.
3. New Service Responsibility
Create:
bookmyslot-ai-service

10. X-Ray Analysis API
Endpoint:
POST /analyse-xray
Request:
multipart/form-data
Input:
file = dental X-ray image
Processing:
Receive Image

      ↓

Save temporary file

      ↓

Load YOLO model

      ↓

Run prediction

      ↓

Format response

      ↓

Return JSON
11. Response Format
Do NOT return raw YOLO output.
Bad:
{
"class_id":2,
"confidence":0.91,
"bbox":[100,200,300,400]
}
Return application-friendly format:
{
"success":true,

"analysis":{

"findings":[

{
"name":"Finding Type 3",

"confidence":91,

"location":{

"x":100,
"y":200,
"width":300,
"height":400

}

}

]

}

}
12. Error Handling
Handle:
Invalid image
Return:
{
"success":false,
"message":"Invalid X-ray image"
}
Model unavailable
Return:
{
"success":false,
"message":"AI service unavailable"
}
